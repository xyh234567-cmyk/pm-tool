# 操作说明(怎么用这套 hub+spoke 系统)

> 这份讲"系统怎么操作"。规则本身看 `核心规则.md`(9 条内核);完整工作流看 `上手说明.md`;结构原理看 `档案结构约定.md`。

## 一、结构速览
- **`设计/` 根 = 方法论 hub(通用之家)**:规则母版(CLAUDE/AGENTS/审查清单)+ 核心规则/精要/上手说明/参考 + 方法论自己的决策记录·路线图。**所有项目共享的东西,只在这改。**
- **`设计/<项目>/` = 项目(专属之家)**:从 hub 拷的规则实例 + 本项目决策记录·路线图 + specs + 代码。**只属于这个项目的东西,只在这改。**

## 二、开新项目(经母版同步器 GUI)
双击 **`母版同步器-GUI.command`** 起 GUI → 顶部切到 **「＋ 新建项目」** 页 → 输项目名 → 创建:
- 自动建 `设计/<项目名>/`,拷入 hub 的 CLAUDE/AGENTS/审查清单母版,生成空的 决策记录/路线图/specs 骨架(同名已存在会拒绝、不覆盖)。
- 完事按提示:**开个新会话**,打开这个项目,对我说"开始",从需求澄清走起。
- (建项目与同步母版统一在这个 GUI 里;旧的 `新建项目.command` 已退役。)

## 三、日常协作
1. **一项目一会话**:换项目=开新会话,别在一个对话里混多个项目(防串味)。
2. 进项目时我会**先声明"当前项目=X、权威文件=…"**,并在动手前打行动戳(防越界/失焦)。
3. 走带闸门的环:**澄清 → 论证 → 设计 → 规格 → 分层生成 → 一项一审 → 沉淀**(细节见 `上手说明.md`)。
4. 代码交 CodeX(读项目里的 `AGENTS.md` + specs);**我只出方案,你执行,我审产出**。
5. commit 由你在评审通过后做(commit = 已评审检查点)。

## 四、维护纪律
- **两套决策记录,别混**:方法论演进 → 写 hub 的;项目口径 → 写项目的。
- **教训单向回流**:新教训若跨项目会复发 → 升级回 **hub 的审查清单**(以后所有项目受益);只对本项目的 → 留项目内。
- **整固**:模块群/项目收尾时,对规则文件合并去重,防臃肿。
- **改了 hub 母版**(CLAUDE/AGENTS/审查清单),老项目不会自动更新;需要时手动把对应改动同步过去(或开新项目时自然带上最新)。

## 五、授权边界(给你的开关)
- 默认:**我只出方案、不碰你项目文件**(动手前打戳,戳里"触碰目标项目"必须为否)。
- 要我直接动手时:**明确说"授权"**——我才在设计区执行,且尽量非破坏(原件先备份)。
- 注:沙盒能建能改、但**删不动**(挂载盘权限),删除/ git 提交始终你本机来。
