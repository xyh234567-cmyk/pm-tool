# 档案结构约定(通用 vs 专属,各有家)

> 解决"多项目发散、串味、不知按谁的规矩"。核心约束:**CodeX/Claude 自动加载的是项目目录里的 CLAUDE.md/AGENTS.md**,所以方法论文件每个项目都得有一份 → 形态是"**一个权威源(hub)+ 各项目一份实例**"。

## 两个家(实际落地结构)
**① 通用之家(hub)= `设计工具/` 文件夹**(本例;不混项目;`CLAUDE.md` 在此,打开 `设计工具/` 时自动加载):
放 `CLAUDE.md`/`AGENTS.md`/`审查清单.md`(母版)+ `核心规则.md`/`档案结构约定.md`/`方法论精要.md`/`上手说明.md`/`设计参考资源.md` + `决策记录.md`/`路线图.md`(**方法论本身**的演进记录)。**通用规则唯一权威源,版本化。**

**② 每个项目(spoke)= 与 hub 同级** `设计/<项目>/`(与 `设计工具/` 平级):
- 从 hub 拷的实例:`CLAUDE.md`、`AGENTS.md`(填第二部分专属)、`审查清单.md`。
- 本项目专属(只此一份、不进 hub):`决策记录.md`、`路线图.md`、`specs/` 或 `_梳理/`、代码。

## 文件归属
| 文件 | 性质 | 家 |
|---|---|---|
| 核心规则 / 方法论精要 / 上手说明 / 设计参考 | 纯通用 | 仅 hub |
| CLAUDE.md | 通用 | hub 母版 → 各项目拷一份 |
| AGENTS.md | 通用 + 专属 | hub 母版 → 项目拷并填专属 |
| 审查清单 | 通用项 + 教训(多数通用) | hub 母版 → 项目拷;**通用教训回流 hub** |
| 决策记录 / 路线图 | 两套并存 | hub 一套(方法论演进)+ 各项目一套(项目口径),互不混 |
| specs/ 或 _梳理/ | 纯专属 | 仅项目 |

## 两条维护纪律
- **教训回流(单向上游)**:新教训若"跨项目会复发"→ 升级回 **hub 的审查清单**(以后所有项目受益);只对本项目的→留项目内。
- **开新项目**:从 hub 拷当前方法论进去当起点。(将来 skill 化可自动注入——见路线图,那是此结构的终极省力形态。)

## 会话纪律(配套,防发散/串味)
- **一项目一会话**(尽量);换项目=新会话,开头先读该项目档案(冷启动索引)。
- 进入某项目工作时,设计方先声明"当前项目=X、权威文件=…",再动手。
