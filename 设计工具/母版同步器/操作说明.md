# 母版同步器 · 操作说明

> 命令行工具,把 hub(`设计工具/`)的规则母版更新辅助下发到各项目(spoke)。
> 设计意图见 `specs/母版同步器/00-总览.md`;本文只讲怎么跑。
> **一句话安全模型:默认 dry-run 只看不写;只有显式加 `--apply` 才动文件,且写前必先备份。**

## 一、在哪跑

进工具目录,用模块方式启动(`-m` 保证 `app` 包能被找到):

```bash
cd 设计工具/母版同步器
python -m app.main            # 默认:通用区下发,dry-run
```

不传 `--hub` 时,工具自动认定脚本上两级目录即 `设计工具/` 为 hub;在别处跑或想指定时加 `--hub <路径>`。

## 二、三种模式(同一次运行只走一条)

| 模式 | 命令 | 同步什么 |
|---|---|---|
| 通用区下发(默认) | `python -m app.main` | `CLAUDE.md`/`AGENTS.md`/`审查清单.md` 的**第一部分(通用区)** |
| 教训上行 | `python -m app.main --lesson-up` | spoke 标了 `[↑]` 的教训 → 汇总提案写入 hub |
| 教训下行 | `python -m app.main --lesson-down` | hub 新增教训 → 找出各 spoke 缺的、下发 |

`--lesson-up` 与 `--lesson-down` 互斥,同时给会报错退出。

## 三、模式一:通用区下发(默认)

把 hub 母版第一部分的更新同步到各 spoke,专属区(第二部分)绝不碰。

```bash
python -m app.main                 # dry-run:列出哪些 spoke 过时、差异在哪,不写
python -m app.main --apply         # 逐文件问 y/N 后写入,写前自动备份
python -m app.main --apply --yes   # 跳过逐文件确认,全部拉取(-y 亦可)
```

dry-run 会在 hub 根生成一份带时间戳的 Markdown 同步报告,可 Git 留痕回看。

## 四、模式二:教训上行(spoke → hub)

把某个项目里沉淀、值得升级为通用的教训提交回 hub。

**第一步,人工标记**:在 spoke 的 `审查清单.md` 第二部分,给要上行的教训标题尾部加 ` [↑]`:

```
- **L14 · 某教训(项目名) [↑]**:教训内容描述。
```

教训编号接着 hub 当前最大 L 号往下编(hub 到 L13,本地新增就写 L14),这是"本地暂定号",并入 hub 后才成官方号。

**第二步,跑工具**:

```bash
python -m app.main --lesson-up           # dry-run:汇总所有 [↑] 候选 + 状态
python -m app.main --lesson-up --apply   # 逐条确认后写入 hub(先备份)
```

工具对每条候选判定状态:**新增**(hub 无此号)、**更新**(hub 有同号但文本不同,给 diff)、**已同步**(文本一致,提示可手删 `[↑]`)、**冲突**(多个 spoke 标了同一 L 号且内容不同 → 只展示不写,人自行协调)。

**写入后**:`[↑]` 标记不自动移除,按报告提示手动删即可(下次下行该号会判为已同步、跳过)。

## 五、模式三:教训下行(hub → spoke)

hub 新增教训后,把各 spoke 缺的补下去。

```bash
python -m app.main --lesson-down           # dry-run:列出每个 spoke 缺哪些 L 号
python -m app.main --lesson-down --apply   # 逐 spoke 逐条确认后按 L 号顺序插入(先备份)
```

只下发 spoke **缺失**的教训;插入位置按 L 号排序(插到正确位置,不是追加末尾)。spoke 自己追加的专属教训完全不动。若 spoke 已有同 L 号(本地暂定号与 hub 号碰撞),跳过并在报告标注,留人工解决。

> 注:hub 改了 spoke **已有**的某条教训,本期**只提示不自动覆盖**(怕冲掉 spoke 的本地微调),需人工处理。

## 六、安全保证(红线)

- **默认 dry-run**:不加 `--apply` 绝不写任何文件。
- **写前必备份**:任何写入前先生成 `<文件名>.bak.<时间戳>`,备份失败即中止、不写。
- **绝不碰专属区**:通用区下发只覆盖第一部分;教训流只动 `审查清单.md` 第二部分,且只增不删。
- **结构异常即停**:受管文件该有 `# 第二部分` 锚点却缺失/重复时,标"结构异常·跳过",不擅自整份当通用区同步。
- **上行不写 spoke、下行不写 hub**:两条流水线方向单一,不会反向污染。
