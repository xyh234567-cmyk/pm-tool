## 2026-06-21 · 协作方式

- **分层规格 + 不写代码**:设计方只产出分层规格文档,代码由 CodeX 生成。理由:职责分离,设计方专注顶层。
- **格式分工**:MD 讲思路、YAML 锁硬契约、Mermaid 画结构、pytest 锁正确性、Git 管历史。理由:各取所长,机器须照搬的部分用确定性格式,降低理解偏差。
- **HTML mockup**:UI 拍板后导出独立 HTML mockup 存 `specs/<项目>/mockups/`,兼作 Jinja2 模板雏形。理由:直观、可留存、降低 UI 走样。
- **质疑力度调高 + 批量提问 + 关键节点复述**(关键节点定义见 CLAUDE.md 行为准则 6)。

## 2026-06-21 · 代码执行环境与文件约定

- **代码生成 agent 锁定 CodeX**(底层模型 DeepSeek)。理由:已有现成环境;CodeX 不自动加载 `CLAUDE.md`,与"设计契约 / 代码规则"两部宪法分离的设计天然契合,无命名冲突。
- **两部宪法分工**:`CLAUDE.md` = 设计阶段(用户↔设计方)协作契约,读者是设计方;`AGENTS.md` = 执行阶段 CodeX 代码生成/修改规则,读者是 CodeX。两者同 repo 共存,CodeX 只读 `AGENTS.md`。
- **全文本化交付**(MD/YAML/Mermaid/HTML mockup)保留为原则:不依赖"读图"能力,任意代码 agent 皆可消费。

## 2026-06-21 · 仓库结构(阶段0 审查确认)

- **每个工具一个顶层目录**:代码在 `设计/<工具名>/app/...`,其规格在 `设计/specs/<工具名>/`。

## 2026-06-21 · 提交时机
