# 方法论精要(北极星 · 一页纸)

> 这是最上层的"为什么"。细节指回:`CLAUDE.md`(设计协作)、`AGENTS.md`(代码规则)、`审查清单.md`(怎么验)、`决策记录.md`(已定口径)、`路线图.md`(范围)。
> 冷启动先读本页,建立全局观,再按 `CLAUDE.md` 顶部清单逐份读。

## 一、核心:把错误赶到最便宜的地方抓,抓一次就变成规则

效率不来自某条规则,来自整台机器在"**左移 + 复利**":每类错误都在"最早、最便宜能抓住它"的关口拦下,漏网的沉淀成规则,下次更早被拦。

| 错误类型 | 最便宜的拦截点 |
|---|---|
| 需求歧义 | 需求澄清(还没动手) |
| 规格错(吃数据形状) | 交手前拿**真实样例干跑** |
| 逻辑错 | **pytest** |
| 接缝/真实数据错 | **真实数据集成跑** |
| 呈现层错(轴向/排序/格式) | 对照 **mockup 人眼看渲染** |
| 漏网的任何一类 | **沉淀成 `审查清单` 规则**,下次更早拦 |

**且只认文件不认对话**——重置、换会话、隔很久都不丢。

## 二、三部宪法,各管一角
- `CLAUDE.md` —— 设计方怎么想、怎么决策。
- `AGENTS.md` —— CodeX 怎么写/改代码。
- `审查清单.md` —— 怎么验 CodeX 的产出 + 已沉淀教训。

## 三、五种载体,各司其职(规格即接口)
MD 讲"为什么/怎么做" · YAML 锁"精确契约" · Mermaid 画结构 · mockup 定"视觉真值" · pytest 锁正确性。

## 四、一个带闸门的环
澄清 → 论证 → 架构/UI → 规格 → **分层生成(一层)→ 审 → 修 → 下一层** → 沉淀。
三条纪律:**不批量**(一层一审)· **审是为确信、不是挑刺** · **沉淀有门槛**(会复发才升级为规则,定期整固防臃肿)。

## 五、底层四条公理(其余都从这长出来)
1. 文件是记忆,对话是草稿(不落盘=没发生)。
2. 质疑要狠,不迎合。
3. 防膨胀,新念头进路线图。
4. 各环境各归其位:沙盒跑逻辑/数据,本机跑运行态/git/人眼验。
