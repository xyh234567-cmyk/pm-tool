# 04 · 页面与交互规格

> 对齐已确认的低保真草图。配色用语义色(红=异常/逾期,黄=预警/临近,绿=正常/完成,灰=未开始/中性)。
> 整体风格:简洁、表格驱动、信息密度优先;服务端渲染,图表用 ECharts。

## 全局布局(base.html)
- **顶部条**:左=标题"项目管理 · 全局监控";右=「扫描 NAS」按钮 + 最近扫描时间(如"最近扫描 06-21 09:12")。
- **导航 tab**:总览 / 延期预警 / 资源撞车 / 数据质检(当前页高亮)。
- 详情页不在 tab 内,从总览行进入,顶部提供"← 返回总览"。

## 颜色口径(统一)
| 语义 | 用途 | 颜色 |
|---|---|---|
| danger 红 | 逾期、项目延期、撞车>阈值、error 质检 | 红 |
| warning 黄 | 临近交付、中风险、进行中、warning 质检 | 黄/琥珀 |
| success 绿 | 已完成任务、低风险 | 绿 |
| neutral 灰 | 未开始、无数据"—" | 灰 |
| 风险标签 | 低=绿底、中=黄底、高=红底 | 按等级 |

## 视图 1 · 项目总览(`/`,overview.html)

### KPI 卡(4 张,横排)
1. 在管项目(中性) 2. 延期(红,>0 时红) 3. 资源撞车(黄,显示"N 人") 4. 7 天内交付(中性)。
- 每张卡可点击 → 对应清单(延期→`/delay`,撞车→`/resource`,临近→`/?deliver=soon`)。

### 筛选条
搜索框(项目名/编号)、阶段下拉、风险下拉、负责人下拉、"仅看延期"开关。**前端过滤**。

### 项目表格(列)
| 列 | 字段 | 说明 |
|---|---|---|
| 编号 | biz_id | |
| 项目 | project_name | 点击行进详情 |
| 阶段 | stage_status | |
| 完成度 | progress_pct | 进度条 + 数字;空显示"—" |
| 风险 | risk_level | 彩色标签 |
| 计划交付 | plan_deliver_date | MM-DD |
| 距交付 | days_to_deliver | 逾期→红"逾期N";临近→黄"N";正常→灰"N";空"—" |

- 表头点击排序(完成度、距交付尤为常用)。
- 行点击 → `/project/{biz_id}`。
- 进度条颜色:>=80 绿、40–79 黄、<40 灰(纯视觉,不改变数据)。

## 视图 2 · 项目详情(`/project/{biz_id}`,detail.html)

### 头部
项目名 + 编号;副行:类型 · 客户 · 总负责人/PM;右侧:状态标签(逾期红/正常)+ 大号完成度%。

### 信息双栏
- 左"基本信息":当前阶段、立项日期、计划交付(逾期标红)、实际交付、风险等级、风险描述、当前主要问题。
- 右"合同与经营":合同状态/编号/金额、已开票、已回款、毛利率;空一律"—"。

### 甘特图(主角)
- ECharts 横道,数据来自 `/api/gantt/{biz_id}`。
- 每行=一个任务;条颜色 done绿/doing黄/todo灰。
- **任务顺序:序号小(早)在最上、晚的在下**,与 `mockups/detail.html` 一致。ECharts 分类轴默认把首项画在底部,**必须 `yAxis.inverse=true`** 翻正,否则顺序会反。
- **红色"今天"竖虚线**;X 轴月份刻度。
- 图例:已完成/进行中/未开始。

### 人员分工表
列:姓名、角色、投入%、参与起止。
- 命中撞车的人 → 姓名旁"撞车"黄标签,点击跳 `/resource` 并定位该人。
- is_external 行可灰显或标"外协"。

### 任务列表(甘特下方)
表格:#、任务名、负责人、计划起止、实际开始、完成度%、状态;延期任务行计划结束标红。

## 视图 3 · 延期预警(`/delay`,delay.html)
- 两段:**项目级延期**(项目、负责人、计划交付、逾期天数、延期原因[任务延期/到期未交付])、**任务级延期**(项目、任务、负责人、计划结束、逾期天数)。
- 按逾期天数降序。行可点链到对应项目详情。
- 空态:"当前无延期 🎉"(纯文本,不用 emoji 也可)。

## 视图 4 · 资源撞车(`/resource`,resource.html)
- 每个撞车人一块:姓名、峰值投入合计(如"110%")、是否"当前正在撞"(includes_today→红"进行中")、冲突时间段、涉及项目(可点链)。
- 排序:当前正在撞优先,其次峰值降序。
- 底部"数据不全"区:列出缺时间/投入、无法判定的人员行,提示去 `/qc`。
- 空态:"未发现资源撞车"。

## 视图 5 · 数据质检报告(`/qc`,qc.html)
- 顶部:最近扫描统计(文件数、新增/更新/跳过、error/warning 数)。
- 表格:严重度(error 红/warning 黄)、类型、来源文件、位置、说明。
- 可按严重度/类型筛选(前端)。
- 空态:"本次扫描未发现数据问题"。

---

## 交互与状态总则
- 所有"空数据"显示"—",不显示 0 或 None;**不因缺字段报错**。
- 数字展示:百分比取整(如 65%),金额千分位。
- 首次未扫描:各页空态引导点"扫描 NAS"。
- 扫描完成:回总览并提示"本次:新增X 更新Y 跳过Z,质检 E错/W警"。
