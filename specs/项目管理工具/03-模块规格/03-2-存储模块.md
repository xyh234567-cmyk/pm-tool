# 03-2 · 存储模块(storage)

## 职责
管理 SQLite:建库建表、幂等写入快照、提供查询接口。不含业务计算(延期/撞车在 analytics)。

## 组成
- `schema.sql`:建表 DDL(见 01 §5)。
- `db.py`:连接获取、初始化(首次运行建表)、事务封装。
- `repository.py`:写入与查询函数。

## 输入/输出
- 输入:`Project`(含 members/tasks)、`QcIssue[]`、`scan_run` 统计。
- 输出:查询返回 dict / dataclass,供 analytics 与 web 使用。

## 依赖
- 标准库 sqlite3;common.models。

## 异常处理
- 首次运行 DB 不存在 → 自动建库建表。
- 写入异常 → 回滚 + 上抛(由 ingest 记 qc 跳过该文件)。
- 查询 DB 为空 → 返回空列表,不报错(页面显示空态)。

## 约定
- 日期一律以 `YYYY-MM-DD` 文本存储,便于字符串比较与 `MAX()`。
- 金额/百分比存原值(REAL),不在存储层做单位换算或四舍五入。
