# 03-1 · 采集解析模块(ingest)

## 职责
扫描 NAS 目录,选取待入库文件,解析每个 xlsx,产出结构化 `Project` 与质检 `QcIssue`,交给 storage 入库。**对源文件只读。**

## 组成
- `scanner.py`:目录扫描、文件名解析、去重选取、编排。
- `parser.py`:单文件解析(逻辑见 02 文档)。
- `template_map.py`:标签字典、表头关键词、区块锚点常量。
- `qc.py`:质检规则(规则表见 02 §7)。

## 输入
- `config.paths.nas_dir`(目录路径)。

## 输出
- `ScanResult`:`{files_total, to_ingest: list[(Project, qc[])], skipped: int, run_stats}`。
- 由编排函数直接调用 storage 写入并返回统计给 web。

## 文件名解析
正则:`^(?P<biz_id>.+?)-(?P<name>.+?)【(?P<date>\d{8})】\.xlsx$`
- 提取 biz_id(文件名版)、name、date(YYYYMMDD→YYYY-MM-DD)。
- 不匹配 → `FILENAME_FORMAT` warning;仍尝试解析内容,快照日期回退文件 mtime 日期。

## 去重选取(同业务同日多文件)
- 以(文件名解析出的 biz_id, date)分组;若一组多文件 → 取文件 mtime 最新者,其余记 `DUP_SNAPSHOT_FILE` warning 并跳过。
- 最终快照主键以**表内 biz_id** 为准(可能与文件名不同)。

## 编排流程(scanner.run)
```
1. 列目录所有 *.xlsx(忽略临时文件 ~$*.xlsx)
2. 解析文件名 → 分组去重 → 得到待处理文件列表
3. for each 文件:
     try:
        project, qc = parser.parse(path, filename_meta)
        storage.upsert_snapshot(project)
        storage.insert_qc(qc)
        累计 inserted/updated
     except Exception as e:
        storage.insert_qc([QcIssue(FILE_OPEN_FAIL/解析异常, ...)])
        累计 skipped；continue   # 不中断整体
4. storage.insert_scan_run(stats)
5. return stats
```

## 依赖
- openpyxl(读 xlsx)。
- storage(写库)。
- common(models、dates、enums、logging)。

## 异常处理
- 单文件任何异常 → 记 error 级 qc + 跳过 + 继续(不得让一个坏文件中断整次扫描)。
- NAS 目录不可访问 → 抛出明确异常,由 web 捕获提示"NAS 路径不可访问",不写库、不崩溃。
- 临时锁文件(`~$`开头)直接忽略。

## 验收要点
- 给定样例文件能正确解析出 4 区块;旧格式样例(业务ID=125)能入库且产出 `BIZ_ID_FORMAT` 与 `ID_FILENAME_MISMATCH`/`FILENAME_FORMAT` 告警。
- 同业务同日两文件只入库一条。
