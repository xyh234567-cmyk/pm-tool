# 03-4 · Web 模块(web)

## 职责
FastAPI 路由 + Jinja2 服务端渲染 + ECharts 图表。把 analytics 数据渲染为 04 文档定义的 5 个视图。

## 硬约束:离线优先,零外部 CDN
- 本工具部署在连 NAS 的本地/内网机器,**可能无外网**。所有前端资源必须本地化,断网也能完整渲染。
- **不引入任何 CSS 框架(含 Tailwind 等 CDN)**;样式用纯 CSS 写在 `static/app.css`,**以 `mockups/` 的两个 HTML 为视觉雏形**沿用其风格。
- **ECharts 必须 vendored 到 `static/echarts.min.js` 本地引用**,不走 CDN。

## 组成
- `routes.py`:路由。
- `deps.py`:注入 config / db 连接。
- `templates/`:`base.html`、`overview.html`、`detail.html`、`delay.html`、`resource.html`、`qc.html`。
- `static/`:`app.css`、`echarts.min.js`(本地或 CDN)、`gantt.js`(详情甘特初始化)。

## 路由表

| 方法 | 路径 | 视图/动作 | 数据来源 |
|---|---|---|---|
| GET | `/` | 项目总览(overview) | `dashboard.get_overview()` + KPI |
| POST | `/scan` | 触发扫描 NAS,完成后重定向回 `/` 并提示统计 | `ingest.scanner.run()` |
| GET | `/project/{biz_id}` | 项目详情 + 甘特 | `dashboard.build_project_detail()` |
| GET | `/delay` | 延期预警清单 | `delay.get_delayed_projects/tasks()` |
| GET | `/resource` | 资源撞车清单 | `resource.get_resource_conflicts()` |
| GET | `/qc` | 数据质检报告 | `storage.get_qc_issues()` + `get_last_scan_run()` |
| GET | `/api/gantt/{biz_id}` | 甘特数据(JSON,供 ECharts) | `dashboard.build_project_detail().tasks` |

> 总览顶部 4 张 KPI 卡分别链接到 `/`(在管)、`/delay`、`/resource`、`/`(临近,带筛选参数 `?deliver=soon`)。

## 页面骨架(base.html)
- 顶部条:标题 + "扫描 NAS"按钮(POST `/scan`)+ 最近扫描时间。
- 主体:KPI 卡区(仅总览页)→ tab 导航(总览/延期预警/资源撞车/数据质检)→ 内容区。
- 配色/布局口径见 04。

## ECharts 甘特(gantt.js)
- 从 `/api/gantt/{biz_id}` 取 JSON:`[{name, plan_start, plan_end, actual_start, color}]` + `{domain_start, domain_end, today}`。
- 用 `custom` series 画横道;X 轴时间域 = [domain_start, domain_end];画 today 竖线(markLine)。
- 颜色:done=绿、doing=黄、todo=灰(取自 CSS 变量或固定色,见 04)。

## 交互
- 总览表头点击排序(前端 JS 即可,数据量小);筛选(阶段/风险/负责人/仅看延期)前端过滤。
- 总览行点击 → `/project/{biz_id}`。
- 撞车页人员"涉及项目"可点链到对应详情。

## 依赖
- FastAPI、uvicorn、Jinja2;analytics、storage、common。

## 异常/空态
- 库为空(从未扫描)→ 各页显示空态卡片:"尚未扫描,请点击右上角『扫描 NAS』"。
- `/scan` 时 NAS 不可达 → 重定向回 `/` 并 flash 错误提示,不 500。
- 详情页 biz_id 不存在 → 404 友好页。

## 验收要点
- 启动后 `/` 可访问;点"扫描 NAS"能拉取样例目录入库并刷新出数据。
- 详情页甘特正确渲染"今天"线与三色条。
- 4 个 KPI 与对应清单页数字一致。
