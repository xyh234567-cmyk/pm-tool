# 00 · 总览(项目管理工具 · 第一期)

> 本目录是分层规格文档,供 AI 编程工具(CodeX)分段消费后直接生成代码。
> 阅读顺序:00 总览 → 01 架构 → 02 数据模型与解析 → 03 模块规格 → 04 页面与交互。
> 文档之间口径必须一致;如发现矛盾,以本系列**后写的、更具体的文档**为准,并回报冲突。

## 1. 一句话定义

一个**本地 Web、单人(PMO)使用**的精简项目监控工具:扫描 NAS 共享文件夹里专人维护的项目 Excel,解析汇总后,提供跨项目的**全局监控**——谁延期、谁有风险、谁资源撞车,并保留每日历史快照。

## 2. 背景与定位

- 用户:公司 PMO 负责人(一人)。公司主营硬件板卡研制。
- 现状:各项目负责人按统一 Excel 模板提供项目信息,由**专人**维护这些 Excel 文件,**每日同步**(有变更就改内容,无变更只改文件名时间戳),统一放在 NAS 共享文件夹。
- 痛点:项目变多后,Excel 看不出**跨项目的全局态势**(延期、风险、资源冲突),也无法留存历史看趋势。
- 本工具解决的是"汇总 + 全局监控 + 分析",**不是**多人协同录入系统。

## 3. 第一期范围(必须做)

仅做"跨项目全局监控看板",含 5 个视图:

1. **项目总览**:表格,一屏看全部项目(阶段、完成度、风险、计划交付、距交付天数、延期红灯),可排序、筛选。
2. **项目详情**:单项目基本信息 + 合同经营 + 甘特图(横道图,带"今天"线)+ 人员分工 + 任务列表。
3. **延期预警**:任务级 & 项目级延期清单。
4. **资源撞车**:按人聚合,时间重叠的项目投入工作量% 叠加 >100% 的预警清单。
5. **数据质检报告**:解析时发现的脏数据(外协/未知人员/一格多名字/必填留空/主键不一致等)告警清单。

数据机制:
- 输入源:NAS 共享文件夹(可配置的本地可访问路径)。
- 采集触发:**手动**点"扫描 NAS"按钮。
- 存储:本地 **SQLite**,按"业务ID + 快照日期"存为**每日快照**,历史保留(供二期趋势)。

## 4. 明确不做(第一期边界 · 防膨胀)

以下进入路线图,第一期**不实现**:

- 趋势视图(完成度/延期数/人员负荷随时间)——但**每日快照照存**,二期直接取历史数据即可。
- 全生命周期流程串联(需求→立项→执行→交付→售后)。
- 需求管理、售后管理子系统。
- 文档归档 / 知识沉淀(PDM 级,最重,单独立项)。
- 多人协同、账号、权限、并发。
- 在系统内**编辑/录入**项目数据(系统对 Excel 只读,不回写)。

## 5. 技术栈(已确认)

| 层 | 选型 | 说明 |
|---|---|---|
| 语言 | Python 3.10+ | 主语言 |
| Web 框架 | FastAPI | 提供路由与服务端渲染入口 |
| 模板渲染 | Jinja2 | 服务端渲染 HTML,无前后端分离 |
| 图表 | ECharts(前端 CDN 或本地静态文件) | 甘特图、状态可视化 |
| Excel 解析 | openpyxl | 读取 .xlsx(只读) |
| 存储 | SQLite(标准库 sqlite3) | 单机单用户,零运维 |
| 运行 | uvicorn | 本地启动,浏览器访问 localhost |

约束:面向"基础开发者自维护",**结构清晰可读、可自调**优先;按"满足功能"定复杂度,不为优雅过度抽象,也不刻意压低复杂度。

## 6. 术语表

| 术语 | 含义 |
|---|---|
| 业务 / 项目 | 一份 Excel = 一个业务 = 一个项目。本文档"项目"与"业务"同义。 |
| 业务ID(主键) | Excel 表内"业务ID"单元格的值,标准格式 `RW{年}-{3位序号}`,例 `RW2026-001`。**全系统唯一主键,以表内单元格为权威源。** |
| 快照(snapshot) | 某个项目在某一天的 Excel 全量解析结果。主键为(业务ID + 快照日期)。 |
| 快照日期 | 取自**文件名**【】中的日期戳(如 `【20260617】` → 2026-06-17)。 |
| 阶段/任务 | Excel"阶段/任务拆解"区每一行,是甘特图数据源。 |
| 投入工作量% | "人员分工"区每人一行的投入百分比,是资源撞车分析的核心字段。 |
| 资源撞车 | 同一人在时间重叠的多个项目里投入%之和 >100%。 |
| 延期 | 见 03 分析模块的精确口径。 |
| 质检 | 解析阶段对脏数据的识别与告警,不静默丢弃、不静默算错。 |

## 7. 二期及以后路线图(仅备忘,不在本期开发)

1. 趋势视图(基于已存的每日快照)。
2. 全生命周期串联与阶段流转。
3. 文档归档与知识沉淀(独立立项评估)。
4. 需求管理、售后管理。

## 8. 正式文件名约定(输入契约)

- 正式文件名:`{业务ID}-{项目名称}【{YYYYMMDD}】.xlsx`,例:`RW2026-001-隔离网关【20260617】.xlsx`。
- 主键以**表内"业务ID"单元格**为准;文件名仅供人阅读与提供快照日期。
- 若表内业务ID 与文件名中的业务ID 不一致 → 以表内为准,并写入**质检报告**。
- 注意:现有样例文件用了旧格式(表内 `125`、文件名 `BIZ-125-...`),属不规范历史数据;正式数据按上述标准。解析器需对"非标准业务ID 格式"产出质检告警(见 02)。
